这学期开始有个想法是在GitHub上搭一个自己的博客，一开始的想法是使用Jekyll，后来由于要做其他项目于是只做了简单的文章发布功能，同时界面实在谈不上好看。临近过年，终于稍微清闲一些，偶然发觉Hexo更容易上手并且效果不错，于是就把博客的内容迁移过来了。

这里只简单记录一些Hexo写作和发布的指令，配置以及其他教程可以Google或者访问Hexo的[文档](https://hexo.io/zh-cn/docs/)

# 前置工作

安装Hexo之前需要安装Git和Node.js。具体的方法可以去官网查看。

OS: Windows 10

# 关联账号

注：似乎不做这一步也可以，在部署的时候会弹出对话框要求登录GitHub，那时候直接输入账号密码也可。

由于我的Hexo博客是挂GitHub的，所以需要关联Hexo和GitHub。

第一步是在GitHub添加ssh，用Git Bash执行指令生成文件id\_rsa.pub，将文件内容添加到GitHub帐号后检测是否联通。

```
ssh-keygen
ssh -T git@github.com
```

之后执行以下指令注明全局用户信息。

```
git config --global user.name "yourname"
git config --global user.email  "youremail@email.com"
```

# 写作

新建文章指令：

```
hexo new [layout] <title>
```

例如：

```
$ hexo new "Hello World.md"
```

# 发布

Hexo提供了快速方便的一键发布功能，只需一条指令就能将网站部署到服务器：`$ hexo generate --deploy`

不过在开始之前，需要先安装hexo-deployer-git：`$ npm install hexo-deployer-git --save`

同时在 `_config.yml` 中修改配置：

```
deploy:
    type: git
    repo: <repository url> #库（Repository）地址
    branch: [branch] #可空缺，GitHub 可自动检测
    message: [message] #可空缺，自定义提交信息
```

# 可能遇到的错误

当新完成一篇md文章时我们可能会发现不能生成静态网页了，可能是因为文章中使用了一些非法字符。

例如该[情况](https://github.com/hexojs/hexo/issues/1228)。

所以，md格式的编写需要留意符号的使用。
